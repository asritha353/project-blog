Incremental ETL Pipeline with Star Schema

Building a complete Data Warehouse using Amazon sales data, PostgreSQL, Python ETL automation, and incremental loading strategy.

By K. Asritha • Final Year Project

Project Overview

This project demonstrates the end-to-end implementation of a Data Warehouse solution using Amazon e-commerce sales data. Raw transactional CSV files were transformed into analytical tables using ETL processing and stored in PostgreSQL using dimensional modelling.

Instead of reloading the full dataset every time, an incremental loading mechanism was implemented using a watermark table.

Source CSV → Python ETL → PostgreSQL Star Schema → SQL Analytics

Dataset Information

  • 128,975 sales records
  • Product category and quantity
  • State / location data
  • Order status and fulfilment details
  • Amount in INR

Warehouse Schema Design

The warehouse follows star schema architecture with one fact table and multiple dimension tables.

  • fact_sales
  • dim_date
  • dim_product
  • dim_location
  • dim_order_status
  • dim_fulfillment
  • etl_watermark

ETL Pipeline Execution

The ETL process performs extraction, transformation, dimension loading, fact loading, and watermark update. 

python main.py

The pipeline successfully loaded all dimension tables and inserted 128,975 rows into fact_sales during the initial run.

Incremental Loading using Watermark

A watermark table stores the latest successfully loaded date. Future executions only load new rows after that date. 

SELECT * FROM etl_watermark;
First Run → Full Load
Next Runs → Only New Rows
Improves Speed & Efficiency
