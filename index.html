Incremental ETL Pipeline with Star Schema

A Data Warehouse Project using Amazon Sales Data

By K. Asritha • April 2026

Introduction

In this project, I built a data warehouse using Amazon e-commerce sales data. The main objective was to create an ETL pipeline that extracts data from CSV files, transforms the raw data, and loads it into PostgreSQL using a star schema design.

I also implemented incremental loading using watermark strategy so that only new records are loaded in future runs.

Architecture

CSV Files → Python → Data Cleaning → PostgreSQL → SQL Analysis

Tools Used:

  • Python
  • Pandas
  • SQLAlchemy
  • PostgreSQL
  • pgAdmin

Star Schema Design

The warehouse was designed using one fact table and multiple dimension tables.

  • fact_sales
  • dim_date
  • dim_product
  • dim_location
  • dim_order_status
  • dim_fulfilment

This design improves reporting and query performance.

ETL Pipeline

Extract

The dataset was loaded from CSV files using Pandas.

Transform

  • Renamed columns
  • Converted date formats
  • Handled missing values
  • Created dimension tables

Load

Dimension tables were loaded first, followed by the fact table into PostgreSQL.

Incremental Loading

Instead of loading all rows every time, I used watermark logic. The last loaded date is stored, and only new records are processed in the next run.

Run 1: Load all records
Run 2: Load only new records
Run 3+: Continue incremental loading

This reduces execution time and improves efficiency.

Business Insights

  • Maharashtra had highest sales
  • April 2022 was peak sales month
  • Set category generated highest revenue
  • B2C orders were more than B2B
  • Cancelled orders were high

Challenges Faced

  • Messy raw column names
  • Missing values in amount column
  • Date conversion issues
  • Foreign key loading order

Conclusion

This project helped me understand ETL pipelines, dimensional modelling, PostgreSQL data warehousing, and incremental loading techniques used in real systems.

It was a practical learning experience in data engineering concepts.
