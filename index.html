Incremental ETL Pipeline with Star Schema

Building a complete PostgreSQL data warehouse using Amazon sales data, Python ETL workflows, and incremental loading using watermark strategy.

By K. Asritha • April 2026 • Data Warehouse / Data Engineering Project

Introduction

Organizations generate large amounts of data every day through sales, customer activity, inventory, and transactions. Raw operational data is useful for day-to-day operations, but it is not optimized for analytical reporting.

To solve this problem, companies build data warehouses that organize historical data in a structured format. In this project, I created a complete data warehouse using Amazon e-commerce sales data and PostgreSQL.

The project includes ETL processing, star schema modelling, SQL analysis, and incremental loading so that future executions process only new data.

Project Objectives

  • Build a complete ETL pipeline using Python
  • Design a star schema warehouse
  • Load cleaned data into PostgreSQL
  • Implement incremental loading
  • Generate business insights using SQL
  • Understand practical data engineering workflow

Star Schema Design

A star schema contains one central fact table connected to multiple dimension tables. This design improves query speed and simplifies reporting.

Fact Table

  • fact_sales – stores transactional measures such as quantity and amount.

Dimension Tables

  • dim_date
  • dim_product
  • dim_location
  • dim_order_status
  • dim_fulfilment
CREATE TABLE dim_product(
 product_key SERIAL PRIMARY KEY,
 sku VARCHAR(50),
 category VARCHAR(50),
 size VARCHAR(20)
);

CREATE TABLE fact_sales(
 sales_key SERIAL PRIMARY KEY,
 product_key INT,
 date_key INT,
 quantity INT,
 amount_inr NUMERIC(12,2)
);

ETL Pipeline

1. Extract

import pandas as pd

df = pd.read_csv(
 "Amazon Sale Report.csv",
 low_memory=False
)

2. Transform

df.columns = df.columns.str.lower()
df.columns = df.columns.str.replace(" ","_")

df['date'] = pd.to_datetime(
 df['date'],
 errors='coerce'
)

df['amount'] = df['amount'].fillna(0)

df = df.drop_duplicates()

3. Load

from sqlalchemy import create_engine

engine = create_engine(
'postgresql://postgres:password@localhost:5432/amazon_dw'
)

df.to_sql(
'staging_sales',
engine,
if_exists='replace',
index=False
)
Dimension tables are loaded first, followed by fact table to maintain foreign key integrity.

Dimension Table Loading

product_df = df[

date_df['month'] = date_df['full_date'].dt.month
date_df['year'] = date_df['full_date'].dt.year

Incremental Loading using Watermark

Instead of loading the entire dataset every run, watermark logic stores the latest processed date. 

CREATE TABLE etl_watermark(
 table_name VARCHAR(50),
 last_loaded_date DATE
);

SELECT last_loaded_date
FROM etl_watermark
WHERE table_name='fact_sales';

new_df = df[
 df['date'] > last_loaded_date
]
First Run → Full Load
Second Run → Only New Rows
Next Runs → Continue Incremental Loading

This reduces processing time and improves scalability.

Business Insights

Top States by Revenue

SELECT state,
SUM(amount_inr) revenue
FROM fact_sales fs
JOIN dim_location dl
ON fs.location_key = dl.location_key
GROUP BY state
ORDER BY revenue DESC;

Monthly Revenue Trend

SELECT month,
SUM(amount_inr)
FROM fact_sales fs
JOIN dim_date dd
ON fs.date_key = dd.date_key
GROUP BY month;
  • Maharashtra generated highest revenue
  • April recorded peak monthly sales
  • Set category was top selling category
  • B2C orders were more than B2B
  • Cancelled orders were significant
