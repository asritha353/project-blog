Incremental ETL Pipeline with Star Schema

Building a complete data warehouse solution using Amazon sales data, PostgreSQL, Python, and efficient incremental loading techniques.

By K. Asritha • April 2026 • Data Engineering / Warehouse Project

Introduction

Modern businesses generate large amounts of transactional data every day. Raw operational data is useful for transactions, but it is not ideal for analytics. To solve this problem, organizations build data warehouses that store structured historical data for reporting and decision making.

In this project, I developed a complete ETL pipeline using Amazon e-commerce sales data. The raw CSV dataset was transformed and loaded into PostgreSQL using a star schema model. I also implemented incremental loading so future executions process only newly added records.

Project Objectives

  • Understand real-world ETL workflow
  • Design a dimensional data warehouse
  • Load data into PostgreSQL
  • Improve performance using incremental ETL
  • Generate business insights through SQL

Star Schema Design

A star schema contains one fact table in the center connected to multiple dimension tables. This model is simple, fast, and widely used in analytics systems.

Fact Table

  • fact_sales – Stores each sales transaction

Dimension Tables

  • dim_date
  • dim_product
  • dim_location
  • dim_order_status
  • dim_fulfilment

This structure supports slicing data by time, product category, region, and order status.

ETL Pipeline

1. Extract

The CSV dataset was imported into Pandas DataFrames. 

import pandas as pd
df = pd.read_csv("Amazon Sale Report.csv")

2. Transform

  • Removed duplicates
  • Handled null values
  • Converted dates
  • Renamed columns
  • Created dimension tables
  • Generated surrogate keys

3. Load

Dimension tables were loaded first, followed by the fact table with foreign key references.

Incremental Loading

Instead of loading the full dataset every time, I used watermark logic. The last loaded date is stored in a control table.

First Run → Load all records
Next Runs → Load only records newer than watermark date 
SELECT last_loaded_date FROM etl_watermark;

WHERE order_date > watermark_date

This reduces processing time and is commonly used in production pipelines.

Business Insights

  • Maharashtra generated highest revenue
  • April 2022 recorded peak monthly sales
  • Set category was top selling product type
  • B2C orders exceeded B2B orders
  • Cancelled orders were significant
